Overview of MarkDiffusion Toolkit

MarkDiffusion is an open-source Python toolkit for generative watermarking of latent diffusion models. As the use of diffusion-based generative models expands, ensuring the authenticity and origin of generated media becomes critical. MarkDiffusion simplifies the access, understanding, and assessment of watermarking technologies, making it accessible to both researchers and the broader community.

Up to date, it unifies 11 state-of-the-art algorithms into a single framework, provides a comprehensive evaluation module with 31 tools and 8 automated pipelines for standardized benchmarking, and includes an intuitive visualization suite to lower the technical barrier. By providing a high-quality, well-documented, and extensible platform, MarkDiffusion promotes reproducible research in the vital area of AI safety, making it an ideal candidate for the JMLR open-source software section.

Key Features

Unified Implementation Framework

MarkDiffusion provides a modular architecture supporting eleven state-of-the-art generative image/video watermarking algorithms of LDMs.

Comprehensive Algorithm Support

Implements 11 watermarking algorithms from two major categories: Pattern-based (Tree-Ring, Ring-ID, ROBIN, WIND, SFW) and Key-based (Gaussian-Shading, PRC, SEAL, VideoShield, GaussMarker, VideoMark).

Visualization Solutions

Custom visualization tools provide clear and insightful views into how different watermarking algorithms operate under various scenarios, helping demystify mechanisms and improve interpretability.

Evaluation Module

With 31 evaluation tools covering detectability, robustness, and impact on output quality, MarkDiffusion offers comprehensive assessment capabilities. It features automated pipelines such as Watermark Detection, Image/Video Quality Analysis, and robustness assessment tools.

How to Install and Run the Software

Google Colab Demo

If you're interested in trying out MarkDiffusion without installing anything, you can use Google Colab to see how it works.

Installation

(Recommended) Install the PyPI package:

conda create -n markdiffusion python=3.11
conda activate markdiffusion
pip install markdiffusion[optional]

(Alternative) Conda-only users can install from conda-forge:

conda create -n markdiffusion python=3.11
conda activate markdiffusion
conda config --add channels conda-forge
conda config --set channel_priority strict
conda install markdiffusion

Note: some advanced features require additional packages that are not available on conda and cannot be included in the release; install them separately if needed.

How to Use the Toolkit

After installation, there are two ways to use MarkDiffusion:

  1. Clone the repository to try the demos or use it for custom development. The MarkDiffusion_demo.ipynb notebook provides detailed demonstrations. Quick example (TR algorithm): 

    import torch
from watermark.auto_watermark import AutoWatermark
from utils.diffusion_config import DiffusionConfig
from diffusers import StableDiffusionPipeline, DPMSolverMultistepScheduler

# Device setup
device = 'cuda' if torch.cuda.is_available() else 'cpu'

# Configure diffusion pipeline
scheduler = DPMSolverMultistepScheduler.from_pretrained("model_path", subfolder="scheduler")
pipe = StableDiffusionPipeline.from_pretrained("model_path", scheduler=scheduler).to(device)
diffusion_config = DiffusionConfig(
    scheduler=scheduler,
    pipe=pipe,
    device=device,
    image_size=(512, 512),
    num_inference_steps=50,
    guidance_scale=7.5,
    gen_seed=42,
    inversion_type="ddim"
)

# Load watermark algorithm
watermark = AutoWatermark.load('TR',
                            algorithm_config='config/TR.json',
                            diffusion_config=diffusion_config)

# Generate watermarked media
prompt = "A beautiful sunset over the ocean"
watermarked_image = watermark.generate_watermarked_media(prompt)
watermarked_image.save("watermarked_image.png")

# Detect watermark
detection_result = watermark.detect_watermark_in_media(watermarked_image)
print(f"Watermark detected: {detection_result}")

  2. Use the PyPI library directly without cloning the repository. The MarkDiffusion_pypi_demo.ipynb notebook provides examples. Quick example: 

    import torch
from markdiffusion.watermark import AutoWatermark
from markdiffusion.utils import DiffusionConfig
from diffusers import StableDiffusionPipeline, DPMSolverMultistepScheduler

# Device
device = "cuda" if torch.cuda.is_available() else "cpu"
print(f"Using device: {device}")

# Model path
MODEL_PATH = "huanzi05/stable-diffusion-2-1-base"

# Initialize scheduler and pipeline
scheduler = DPMSolverMultistepScheduler.from_pretrained(MODEL_PATH, subfolder="scheduler")
pipe = StableDiffusionPipeline.from_pretrained(
    MODEL_PATH,
    scheduler=scheduler,
    torch_dtype=torch.float16 if device == "cuda" else torch.float32,
    safety_checker=None,
).to(device)

# Create DiffusionConfig for image generation
image_diffusion_config = DiffusionConfig(
    scheduler=scheduler,
    pipe=pipe,
    device=device,
    image_size=(512, 512),
    guidance_scale=7.5,
    num_inference_steps=50,
    gen_seed=42,
    inversion_type="ddim"
)

# Load Tree-Ring watermark algorithm
tr_watermark = AutoWatermark.load('TR', diffusion_config=image_diffusion_config)
print("TR watermark algorithm loaded successfully!")

# Generate watermarked image
prompt = "A beautiful landscape with mountains and a river at sunset"
watermarked_image = tr_watermark.generate_watermarked_media(input_data=prompt)

# Save image
watermarked_image.save("watermarked_image.png")
print("Watermarked image generated!")

# Detect watermark in the watermarked image
detection_result = tr_watermark.detect_watermark_in_media(watermarked_image)
print("Watermarked image detection result:")
print(detection_result)

Point-by-Point Mapping of MarkDiffusion to the MLOSS Review Criteria

The impact on the machine learning community

  • The novelty, breadth, and significance of the contribution. MarkDiffusion is a unified, extensible toolkit for generative watermarking of latent diffusion models, consolidating state-of-the-art algorithms, evaluation, and visualization into a single framework.
  • Evidence of an active user community. The repository currently has 177 stars and is under active development, with continuous updates that add new algorithms and evaluation tools.
  • The openness of the project. The project has a public Github repository. The project already has issue/PR interactions. To better engage with the community, we have adopted the Apache-2.0 license and formalized GitHub processes, including a Code of Conduct, contributing guidelines, a security policy, and standardized templates for pull requests and issues.
  • Comparison to previous related implementations. For toolkits targeting generative watermarking for LDMs, MarkDiffusion is the first work (pioneering). Comparisons to related works in adjacent areas are provided in the paper's Appendix C.

User documentation

  • Installation instructions. We provide detailed installation instructions through a multilingual GitHub README and through Read the Docs site. We also released an engineering-standard PyPI package—users can install via pip install markdiffusion without manually setting up dependencies. In addition, we provide a conda-forge package for conda-only environments.
  • Tutorials & Non-trival examples. We provide rich tutorials for new users, including an online Google Colab demo, and two local Jupyter notebooks (MarkDiffusion_demo.ipynb and MarkDiffusion_pypi_demo.ipynb). These tutorials demonstrate typical use cases: generating watermarked images/videos with supported algorithms, detection, visualization of watermarking mechanisms, and comprehensive evaluation/benchmarking.
  • Full documentation of the API. Full documentation of the API is provided on the Read the Docs site.

Openness to new developers / Ease of contribution / sustainability

  • How easy is it for new contributors to participate / contribute & The quality of the developer documentation. We provide a “How to contribute” section in Read the Docs site to guide new contributors, and we provide API references to reduce the barrier for extensions and integration.
  • The clarity of software design.. We present the overall class design of the repository using a use-case diagram in the paper's Appendix A.

Implementation and adoption of software development best practices

  • Extensive unit and integration testing. We provide extensive unit and integration testing, with 658 unit tests covering all key functions. Code coverage is 95%. Here are the coverage report and the result report exported directly via pytest.
  • Continuous integration and testing. We provide continuous integration and testing via GitHub Actions. Given the large codebase and tests that involve downloading/loading models, CI dynamically selects the test scope based on changed files to balance rigor and efficiency (core skeleton changes trigger the full algorithm suite; per-algorithm changes trigger targeted tests; evaluation changes trigger evaluation tests).
  • The breadth of platforms it can be used on. MarkDiffusion runs on common open-source OS environments (e.g., Linux) and standard Python tooling (pip/conda environments). We provide online Google Colab demo for quick testing.
  • The freedom of the code (lack of dependence on proprietary software). The toolkit is open-source (Apache-2.0) and does not require proprietary software to run.
